# SNS投稿作成ツール - 要件定義

## 概要
ユーザーが投稿した文章を元に、バズる可能性の高いSNS投稿案を自動生成するツール

## 対象プラットフォーム
- 𝕏（旧Twitter）: 10個
- Threads: 10個

## 主な機能

### 1. 投稿案生成機能
- **入力**: ユーザーが作成した元となる文章
- **出力**: バズる可能性の高い投稿案 合計20個（𝕏用10個、Threads用10個）
- **生成ロジック**: Claude Code上でClaudeが直接生成

### 2. 入力方法
- **実装形態**: Claude Code（このチャット環境）
- **入力方法**:
  - ファイルアップロード
  - チャットへ直接コピペ

### 3. 出力方法
- **出力形式**:
  - MDファイル（ユーザー確認用）
  - TSVファイル（コピペ用）
- **保存場所**: `03_OUTPUT/` ディレクトリ
- **ファイル命名規則**: `YYYYMMDD_[内容].md` / `YYYYMMDD_[内容].tsv`
  - 例: `20251203_投稿案.md`, `20251203_投稿案.tsv`

## バズる要素の定義（バズの設計書に基づく）

### 1. 稀有性（最重要）
「全員が知っているが、実際に出会うことはない」要素を武器にする
- 例：養蜂家→「10万匹の蜂と暮らす男」というキャッチーな表現
- 例：一夫多妻→内情は知らない関係性にフォーカス

### 2. 最強の方程式「稀有性 × 一般質問」
非日常的な存在（稀有な人）に、誰もが共感できる日常的な質問をぶつける
- 視聴者は「自分だったらどうする？」と自分ごと化せずにはいられなくなる
- 好奇心が爆発する構造

### 3. プロセスエコノミー
完成されたコンテンツより、未完成なリアルが求められる
- 「弱者」からのスタート（例：大赤字、ど素人）
- 達成可能か不明な目標設定
- リアルタイム性（過去のドキュメンタリーではなく、今進行中の物語）

### 4. 冒頭2秒の感情トリガー（必須）
無心でスワイプする視聴者の心を動かすため、冒頭で感情を強く刺激する
- 共感性羞恥（例：街中でのナンパ動画）
- 恐怖・緊張（例：プロポーズ直前の男性）
- 驚き・ギャップ（例：強面の男性が子犬にデレデレ）
- 共感・自分ごと化（例：新卒でメンタル崩壊した話）
- 感動・応援（例：100kgのデブが痩せる挑戦）
- 怒り・不快（例：悪質なSNS詐欺を暴露）

### 5. PR感のないPR
広告だと気づかれないまま、プロダクトの価値を刷り込む
- NG例：サービスの利点を説明する
- OK例：物語を醸し、非顧客層にもリーチする

### 6. 異常値を出す（AI時代の生存戦略）
AIは「平均点」を出す。人間の仕事は「異常値」を出すこと
- 誰も見たことがないような尖ったアイデア
- Googleにも答えがない、AIにも予測できない、あなただけの「異常値」
- 日常のあらゆる出来事を「どうすればバズるか」というレンズで見る癖をつける

### 技術的制約
- 文字数最適化（𝕏: 280文字厳守）
- **ハッシュタグ**: 使用しない
- **絵文字**: 使用しない

## 投稿案の多様性
- **トーン**: 混在させる
  - 真面目
  - カジュアル
  - ユーモラス
  - 共感型（自分ごと化）
  - 挑発的
  - 質問形式（稀有性 × 一般質問）
  - プロセス型（弱者からのスタート）
  - 物語型（リアルタイムドラマ）
  - など、様々なアプローチを含める

- **感情トリガー**: 各投稿案に必ず1つ以上の感情トリガーを埋め込む
  - 共感性羞恥、恐怖・緊張、驚き、共感、感動、怒り・不快のいずれか

## 技術スタック
- **実行環境**: Claude Code
- **生成AI**: Claude (Sonnet 4.5)
- **出力フォーマット**: Markdown, TSV

## ユーザーストーリー
1. ユーザーは元となる文章をClaude Codeへファイルアップロードまたはコピペする
2. Claudeは文章を分析し、以下を抽出する：
   - 稀有性（ユニークな属性、経験、視点）
   - 一般質問との組み合わせ可能性
   - プロセスエコノミーの要素（弱者性、未完成性、成長過程）
   - 感情トリガーの候補
   - 異常値となる可能性のあるアイデア
3. Claudeは𝕏向けに最適化された投稿案を10個生成する
   - 各投稿案に感情トリガーを明記
   - 280文字以内に収める
   - バズの法則（稀有性、プロセスエコノミー等）を適用
4. Claudeは Threads向けに最適化された投稿案を10個生成する
   - 𝕏より若干長めの展開も可能
   - ストーリー性を重視
5. Claudeは生成した投稿案をMDファイルとTSVファイルで `03_OUTPUT/` に保存する
6. ユーザーは生成された投稿案を確認・選択する

## ディレクトリ構成
```
251203_SNS Creater/
├── Claude.md           # 要件定義（本ファイル）
├── バズの設計書.pdf    # 参考資料
├── 01_INPUT/           # 入力ファイル格納用（オプション）
├── 02_WORK/            # 作業用（オプション）
└── 03_OUTPUT/          # 出力ファイル格納用
    ├── YYYYMMDD_投稿案.md
    └── YYYYMMDD_投稿案.tsv
```

## 出力ファイル仕様

### MDファイル
- 見やすさ重視
- プラットフォーム別にセクション分け
- 各投稿案に以下を明記：
  - 番号
  - 投稿内容
  - 文字数
  - 適用したバズの法則（稀有性、プロセスエコノミー等）
  - 感情トリガー（共感性羞恥、驚き等）

### TSVファイル
- コピペしやすいタブ区切り形式
- カラム構成: `No.` `Platform` `投稿内容` `文字数` `バズの法則` `感情トリガー`
