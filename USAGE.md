# 使用方法ガイド

SNS投稿作成ツールの詳しい使い方を説明します。

## クイックスタート

### 1. スラッシュコマンドで生成（最も簡単）

```
/sns
```

コマンド実行後、元となる文章を入力してください。

**新機能**: オリジナリティチェック機能が追加されました。生成後、元の文章と類似度が高い投稿案があれば自動的に指摘し、再生成を提案します。

### 2. 直接依頼で生成

Claude Codeで以下のように依頼：

```
以下の文章を元に、𝕏とThreads用の投稿案を各10個生成してください。

[元となる文章をここに貼り付け]
```

## 詳細な使い方

### ステップ1: 元となる文章を準備

#### 方法A: テンプレートを使用
1. `01_INPUT/input_template.md` を開く
2. テンプレートをコピーして新しいファイルを作成
3. 元となる文章を記入

#### 方法B: 直接入力
- Claude Codeのチャットに直接文章を貼り付け

### ステップ2: 投稿案を生成

#### 方法A: スラッシュコマンド
```
/sns
```

このコマンドは以下を自動的に実行します：
- 文章の分析（稀有性、感情トリガー、バズの要素を抽出）
- 𝕏用投稿案10個の生成
- Threads用投稿案10個の生成
- 各投稿案にバズの法則と感情トリガーを明記
- **オリジナリティチェック**（元の文章と類似度が高い投稿案を自動検出）
- NG投稿案の再生成提案

#### 方法B: 直接プロンプト
以下のようにClaudeに依頼：

```
以下の文章から、バズるSNS投稿案を生成してください：
- 𝕏用: 10個（280文字以内）
- Threads用: 10個（ストーリー性重視）

バズの法則（稀有性、プロセスエコノミー、感情トリガー）を適用してください。

[元となる文章]
ここに文章を貼り付け
```

### ステップ3: オリジナリティチェック結果を確認

生成後、自動的にオリジナリティチェックが実行されます。

**チェック内容:**
- 元の文章と同じ表現を使っている投稿案を特定
- 50%ルール（少なくとも50%は新しい表現）を守れているか確認
- 類似度が高い投稿案があれば番号と理由を明示

**NG判定された場合:**
- 該当する投稿案の番号が表示されます
- 「この投稿案は元の文章と類似度が高いため、再生成しますか？」と確認されます
- 「はい」と答えれば、該当投稿案のみ再生成されます

### ステップ4: 投稿案を確認

生成された投稿案を確認し、以下をチェック：
- **文字数**: 𝕏用は280文字以内になっているか
- **バズの法則**: 各投稿案に適切なバズの法則が適用されているか
- **感情トリガー**: 感情トリガーが明確に示されているか
- **多様性**: 様々なトーンやアプローチが含まれているか
- **オリジナリティ**: 元の文章をそのまま使いすぎていないか

### ステップ5: 投稿案を保存

#### 方法A: Claudeに依頼（推奨）
```
これらの投稿案をMDファイルとTSVファイルとして03_OUTPUT/に保存してください。
```

#### 方法B: Pythonスクリプトを使用
1. 投稿案をJSON形式で保存
2. 以下のコマンドを実行：
```bash
python save_posts.py 02_WORK/posts.json
```

### ステップ6: 投稿案を活用

生成されたファイルを開いて、投稿案を確認：
- **MDファイル**: 見やすく整形された一覧
- **TSVファイル**: Excelやスプレッドシートにコピペ可能

## 実践例

### 例1: 起業ストーリーから投稿案を生成

**入力文章:**
```
3年前、月収20万円のサラリーマンだった私が、今では年商3億円の会社を経営しています。
きっかけは、上司に理不尽に怒られた日でした。
```

**生成コマンド:**
```
/generate-sns-posts
```

**結果:**
- 𝕏用投稿案10個（稀有性、プロセスエコノミー、感情トリガー適用）
- Threads用投稿案10個（ストーリー性重視）

### 例2: 教育問題から投稿案を生成

**入力文章:**
```
日本の教育制度は、100年前からほとんど変わっていません。
みんな同じ制服を着て、同じ教室で、同じ内容を学ぶ。
個性を殺し、従順な労働者を育成するシステムが、今も続いています。
```

**生成コマンド:**
```
/generate-sns-posts
```

**結果:**
- 社会問題を問いかける投稿案
- 共感と怒りを喚起する投稿案
- 議論を促す質問形式の投稿案

## Tips: より良い投稿案を生成するコツ

### 1. 稀有性を強調
元となる文章に、あなたの独自の経験や視点を含める

**NG例:**
```
起業は大変です。でも、頑張れば成功できます。
```

**OK例:**
```
3年前、借金500万円からスタート。今では年商3億円。
失敗を20回繰り返して、やっと成功しました。
```

### 2. 具体的な数字を入れる
抽象的な表現より、具体的な数字の方がバズりやすい

**NG例:**
```
たくさんの本を読んで、人生が変わりました。
```

**OK例:**
```
1年間で365冊の本を読んだら、年収が1.5倍になりました。
```

### 3. 感情を込める
喜び、怒り、悲しみ、驚きなどの感情を含める

**NG例:**
```
新しいビジネスを始めました。
```

**OK例:**
```
上司に「お前には才能がない」と言われ、悔しくて眠れなかった。
その怒りが、今の成功の原動力になりました。
```

### 4. プロセスを見せる
完成品より、未完成なリアルを見せる

**NG例:**
```
成功しました！
```

**OK例:**
```
今日で起業100日目。まだ売上ゼロ。
でも、諦めません。毎日の挑戦を記録していきます。
```

## トラブルシューティング

### Q: 投稿案が生成されない
A: 元となる文章が短すぎる可能性があります。200文字以上を目安に入力してください。

### Q: 投稿案がバズの法則に沿っていない
A: `/generate-sns-posts` コマンドを使用するか、プロンプトで明確に「バズの法則を適用してください」と指示してください。

### Q: 𝕏用投稿案が280文字を超えている
A: Claudeに「280文字以内に修正してください」と依頼してください。

### Q: 保存がうまくいかない
A: `save_posts.py` スクリプトが正しく配置されているか確認してください。
```bash
python save_posts.py
```
でテスト実行できます。

## 応用テクニック

### 1. 特定のトーンに絞る
```
/generate-sns-posts
※ユーモラスなトーンのみで生成してください
```

### 2. 特定のプラットフォームのみ生成
```
𝕏用の投稿案のみ20個生成してください
```

### 3. 追加の投稿案を生成
```
先ほどの投稿案に加えて、さらに10個生成してください
```

### 4. 投稿案の修正
```
投稿案3を、もっと挑発的なトーンに修正してください
```
